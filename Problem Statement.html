Car Data API – Spring Boot

In this challenge, you are part of a team that is building a car data API. One requirement is for a REST API service to manage car data details using Spring boot. You will need to add functionality to add and show car data as well as list all the products in the system. The team has come up with a set of requirements, including the API format, response codes, and the structure for the queries you must implement. The definitions and detailed requirements list follow. You will be graded on whether your application performs data retrieval and manipulation based on given use cases exactly as described in the requirements.

Each product has the following structure:

  • id: the unique ID of the car data [int]
  • model: the name of the model [String]
  • manufacturedYear: the manufactured year of the model [int]
  • company: the company name [String]
  • bodyType: the type of the model [String]
  • price: the price of the model [int]
  • fuelType: the fuel type of the body type [String]
  • seatingCapacity: the seating capacity of the body type [int]

Sample product JSON:

        {
            "model": "City",
            "manufacturedYear": 2024,
            "company": "Honda",
            "bodyType": "sedan",
            "price": 2000000.0,
            "fuelType": "petrol",
            "seatingCapacity": 5
        }

API Endpoints:

  1. Adding a new car data entry:

    • Endpoint: POST /carData/add
    • Request Body: JSON object containing car data details
    • Response Code: 201 if successful, 400 if data validation fails

  2. Retrieving all car data entries:

    • Endpoint: GET /carData/list
    • Response Code: 200 if successful, 400 if no data exists

  3. Updating car data information:

    • Endpoint: PATCH /carData/update/{id}
    • Request Body: JSON object containing updated car data details
    • Response Code: 200 if successful, 400 if no data provided, 404 if ID not found

  4. Deleting a car data entry by ID:

    • Endpoint: DELETE /carData/delete/{id}
    • Response Code: 200 if successful, 404 if ID not found
